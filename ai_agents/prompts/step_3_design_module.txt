## ⚡ THE THREE UNBREAKABLE LAWS OF CIRCUIT DESIGN

### LAW 1: ONE PIN = ONE NET (Physics Constraint)
A physical pin can only exist at one voltage potential. Connecting it to different nets would create a short circuit and destroy the device.

**Implementation:**
1. Create `pinNetMapping` BEFORE making connections
2. Once a pin is assigned to a net, it STAYS with that net
3. All connections to that pin MUST use the same net

### LAW 2: NO FLOATING COMPONENTS (Manufacturing Constraint)
Every component must be fully connected and serve a purpose. PCB fabrication requires complete netlists.

**Requirements:**
- Every component must have ALL pins connected
- No "spare" or "optional" components
- No unconnected test points or debug ports

### LAW 3: COMPLETE CURRENT PATHS (Electrical Constraint)
Current must flow in complete loops from power through components to ground.

**Requirements:**
- Power distribution to all active components
- Ground return paths for all circuits
- Bypass capacitors for all ICs

## CRITICAL: HANDLING EXTERNAL INTERFACES AND SPECIAL COMPONENTS

### EXTERNAL INTERFACES (Connectors, Control Signals)
External interfaces are pins that connect to the outside world. These MUST be included in pinNetMapping but handled specially:

**For External Control Signals (ENABLE, etc.):**
```javascript
// In components:
{"refDes": "J4", "type": "connector", ...}

// In pinNetMapping:
"J4.1": "ENABLE"  // External enable signal

// In connections - DON'T leave single-ended!
// Connect to internal circuitry OR create a pull-up/down:
connections.push({
  "net": "ENABLE",
  "points": ["J4.1", "R10.1"]  // Connect to pull-up resistor
})
connections.push({
  "net": "VCC",
  "points": ["R10.2", "U1.14"]  // Pull-up to VCC
})
```

**For External Power (UART_VCC, VDD_SWITCHED):**
```javascript
// Connect to internal power distribution:
connections.push({
  "net": "UART_VCC",
  "points": ["J3.5", "C10.1", "U5.8"]  // Power to UART chip + bypass cap
})
```

### SWITCHES - ALWAYS CONNECT ALL TERMINALS
Switches MUST have all terminals connected:
```javascript
// 3-terminal switch (SPDT):
"SW1.1": "SW1_NO",     // Normally open
"SW1.2": "GND",        // Common - ALWAYS connect to GND or reference
"SW1.3": "SW1_NC"      // Normally closed

// Create complete connections:
connections.push({
  "net": "GND",
  "points": ["SW1.2", "C1.2", "U1.4"]  // Common to ground
})
connections.push({
  "net": "SIGNAL_IN",
  "points": ["SW1.1", "R1.1"]  // NO to signal
})
```

### POTENTIOMETERS - ALL THREE PINS CONNECTED
Potentiometers MUST have all 3 pins connected:
```javascript
// 3-terminal potentiometer:
"RV1.1": "VCC",        // Top terminal
"RV1.2": "ADC_INPUT",  // Wiper - connects to ADC or op-amp
"RV1.3": "GND"         // Bottom terminal

// Create complete connections:
connections.push({
  "net": "VCC",
  "points": ["RV1.1", "U1.8", "C1.1"]
})
connections.push({
  "net": "ADC_INPUT",
  "points": ["RV1.2", "U2.3", "C5.1"]  // Wiper to ADC with filter cap
})
connections.push({
  "net": "GND",
  "points": ["RV1.3", "C1.2", "U1.4"]
})
```

## CRITICAL VALIDATION CHECKLIST
Before outputting your design, verify:

1. ✅ **pinNetMapping exists and is complete**
   - Every component pin that's used appears in pinNetMapping
   - Each pin maps to exactly one net

2. ✅ **NO single-ended nets**
   - Every net connects to at least 2 pins
   - External signals have pull-ups/downs or connections
   - Switch commons connect to ground
   - Potentiometer wipers connect to circuits

3. ✅ **Connections are properly formatted**
   - Each connection has a "points" array with 2+ elements
   - The "net" field matches pinNetMapping entries

4. ✅ **No floating components**
   - Every component appears in connections
   - All pins are accounted for

5. ✅ **Power and ground connected**
   - All ICs have power pins connected to appropriate rails
   - All ICs have ground pins connected
   - Bypass capacitors present for all ICs

6. ✅ **Auxiliary IC pins connected (No Naked Auxiliaries)**
   - Every IC pin that is NOT power/ground/interface MUST connect to a passive component
   - Filter pins (FILT, FLT, COMP, MFLT) → capacitor to GND
   - Offset pins (OFS, OFSA, OFSB, TRIM) → capacitor or resistor network
   - Bias pins (BIAS, REF, SET, ISET) → resistor to set operating point
   - Soft-start pins (SS, RT, CT) → capacitor or resistor
   - Bootstrap pins (BOOT, BST) → capacitor to switching node
   - Average/RMS pins (AVG, OAVG, SUMR, SUMI) → filter capacitor to GND
   - Buffer pins (IBUF, OBUF, IHP, OHP) → bypass capacitor or signal chain
   - If any IC pin has a net with only 1 connection point, ADD the missing passive!

❌ **ERROR: Single-ended external signal**

