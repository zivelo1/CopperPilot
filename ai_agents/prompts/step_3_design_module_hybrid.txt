## THE THREE UNBREAKABLE LAWS OF CIRCUIT DESIGN

### LAW 1: ONE PIN = ONE NET (Physics Constraint)
A physical pin can only exist at one voltage potential. Connecting it to different nets would create a short circuit and destroy the device.

**Implementation:**
1. Create `pinNetMapping` BEFORE making connections
2. Once a pin is assigned to a net, it STAYS with that net
3. All connections to that pin MUST use the same net

### LAW 2: NO FLOATING COMPONENTS (Manufacturing Constraint)
Every component must be fully connected and serve a purpose. PCB fabrication requires complete netlists.

**Requirements:**
- Every component must have ALL pins connected
- No "spare" or "optional" components
- No unconnected test points or debug ports

### LAW 3: COMPLETE CURRENT PATHS (Electrical Constraint)
Current must flow in complete loops from power through components to ground.

**Requirements:**
- Power distribution to all active components
- Ground return paths for all circuits
- Bypass capacitors for all ICs

## CRITICAL: COMPONENT RATING SELECTION (GENERIC FOR ALL CIRCUITS)

**COMPONENT RATINGS MUST MATCH APPLICATION REQUIREMENTS**

This is a CRITICAL requirement that applies to ALL circuit types - from simple LED blinkers to high-power amplifiers. Components that are structurally connected correctly but have inadequate voltage/current/power ratings will FAIL or be DESTROYED.

### UNIVERSAL RATING SELECTION RULES:

**1. MOSFET/Transistor Voltage Rating:**
- ALWAYS check the output voltage or bus voltage in the requirements
- MOSFET Vds rating MUST be at least 1.5x the peak voltage (2x for high voltage)
- Example: For 100V peak output -> use MOSFETs rated 150V minimum (200V preferred)
- Example: For 360Vpp output -> peak is 180V -> use MOSFETs rated 400V+ (500V preferred)

**Common MOSFET Selection Guide:**
| Application Voltage | Minimum Vds Rating | Example Parts |
|---------------------|-------------------|---------------|
| <=24V (low voltage) | 40-60V | IRFZ44N, IRF3205, BS170 |
| 25-50V (medium-low) | 80-100V | IRF540N, IRF530, IRFB4110 |
| 51-100V (medium) | 150-200V | IRFP260N, IRFP264 |
| 101-200V (medium-high) | 300-400V | IRF740, FQP13N50 |
| 201-300V (high) | 450-600V | IRFP450, STW34NM60, IPP60R099C6 |
| >300V (very high) | 600V+ | IPP60R099C6, SPW47N60C3, IGBTs |

**2. Capacitor Voltage Rating:**
- Capacitor voltage rating MUST be at least 1.5x working voltage (2x for electrolytics)
- Example: For 48V rail -> use 100V rated capacitors minimum
- Example: For 180V rail -> use 250V or 400V rated capacitors
- ALWAYS specify voltage rating in specs: "100uF/100V" not just "100uF"

**3. Power/Current Considerations:**
- For high-power designs (>10W), consider:
  - MOSFET Rds(on) for power dissipation
  - Package thermal ratings (TO-220 > SOT-23)
  - Heat sinking requirements
  - Current sense resistor power ratings (2W, 5W packages for high current)

**4. Frequency Considerations:**
- For high-frequency designs (>100kHz):
  - Use MOSFETs with fast switching (low Qg)
  - Use RF-rated components
  - Consider parasitic inductance in layout

## CRITICAL: IC PINOUT VERIFICATION METHODOLOGY

**NEVER ASSUME PIN FUNCTIONS FROM PIN NUMBERS ALONE**

Different ICs have different pinout configurations. You MUST verify the actual pin functions for EVERY IC you use.

### UNIVERSAL VERIFICATION PROCESS:

**Step 1: Identify Pin Function Categories**
For EVERY IC, determine which pins are:
- **INPUT pins**: Where signals/power come INTO the IC
- **OUTPUT pins**: Where signals/power come OUT of the IC
- **POWER pins**: Supply voltage (VCC, VDD, VIN, etc.)
- **GROUND pins**: Ground reference (GND, VSS, etc.)
- **CONTROL pins**: Enable, adjust, feedback, shutdown, etc.

**Step 2: Apply Electrical Logic Validation**
For each pin connection, ask yourself:
- Does this connection make electrical sense?
- Is power flowing in the correct direction?
- Are signals going from source to destination logically?

**Step 3: Cross-Check Critical ICs**
For voltage regulators, verify:
- Which pin receives UNREGULATED input voltage?
- Which pin delivers REGULATED output voltage?
- Which pin connects to ground reference?
- Which pin is for adjustment/feedback (if applicable)?

For amplifiers/op-amps, verify:
- Which pins are inputs (inverting/non-inverting)?
- Which pin is the output?
- Which pins are power supply (V+, V-)?

### EXAMPLES OF CORRECT PINOUT HANDLING:

**Three-Terminal Voltage Regulator:**
```
# WRONG - Don't assume pin 2 is always ground!
# Some regulators: Pin 1=IN, Pin 2=GND, Pin 3=OUT (like 78xx)
# Other regulators: Pin 1=ADJ, Pin 2=OUT, Pin 3=IN (like LM317)

# CORRECT - Verify for the SPECIFIC IC:
# LM317: Pin 1=ADJUST, Pin 2=OUTPUT, Pin 3=INPUT
# 7805:  Pin 1=INPUT, Pin 2=GND, Pin 3=OUTPUT
```

### CRITICAL REMINDER:
**Pin numbers are just labels. Pin FUNCTIONS determine how to connect them.**

### STEP 3: BUILD pinNetMapping FIRST
This is your CONTRACT with the electrical laws:
```javascript
pinNetMapping = {
  "U1.7": "VCC",     // U1 power pin
  "U1.4": "GND",     // U1 ground pin
  "C1.1": "VCC",     // Bypass cap positive
  "C1.2": "GND",     // Bypass cap negative
  // Add EVERY pin you'll use
}
```

### STEP 4: CREATE CONNECTIONS WITH PROPER POINTS ARRAYS
**CRITICAL: Every connection MUST have a populated points array!**

Group all pins that share the same net:
```javascript
connections.push({
  "net": "VCC",
  "points": ["U1.7", "C1.1", "R1.1", "U2.14"]  // ALL pins on this net
})
```

## CRITICAL: HANDLING EXTERNAL INTERFACES AND SPECIAL COMPONENTS

### EXTERNAL INTERFACES (Connectors, Control Signals)
External interfaces are pins that connect to the outside world. These MUST be included in pinNetMapping but handled specially:

**For External Control Signals (ENABLE, etc.):**
```javascript
// In components:
{"refDes": "J4", "type": "connector", ...}

// In pinNetMapping:
"J4.1": "ENABLE"  // External enable signal

// In connections - DON'T leave single-ended!
// Connect to internal circuitry OR create a pull-up/down:
connections.push({
  "net": "ENABLE",
  "points": ["J4.1", "R10.1"]  // Connect to pull-up resistor
})
connections.push({
  "net": "VCC",
  "points": ["R10.2", "U1.14"]  // Pull-up to VCC
})
```

### SWITCHES - ALWAYS CONNECT ALL TERMINALS
Switches MUST have all terminals connected:
```javascript
// 3-terminal switch (SPDT):
"SW1.1": "SW1_NO",     // Normally open
"SW1.2": "GND",        // Common - ALWAYS connect to GND or reference
"SW1.3": "SW1_NC"      // Normally closed
```

### POTENTIOMETERS - ALL THREE PINS CONNECTED
Potentiometers MUST have all 3 pins connected:
```javascript
// 3-terminal potentiometer:
"RV1.1": "VCC",        // Top terminal
"RV1.2": "ADC_INPUT",  // Wiper - connects to ADC or op-amp
"RV1.3": "GND"         // Bottom terminal
```

## COMMON ERRORS TO AVOID

**ERROR: Single-ended external signal**
```json
// WRONG:
{"net": "ENABLE", "points": ["J1.1"]}  // Only one connection!
```
**FIX: Add pull-up or internal connection**
```json
{"net": "ENABLE", "points": ["J1.1", "R10.1", "U1.5"]}
```

**ERROR: Inadequate MOSFET voltage rating**
```json
// WRONG for 360Vpp application:
{"refDes": "Q1", "type": "mosfet", "value": "IRF540N"}  // Only 100V!
```
**FIX: Use properly rated MOSFET**
```json
{"refDes": "Q1", "type": "mosfet", "value": "IRFP450", "specs": {"Vds": "500V"}}
```

**ERROR: Inadequate capacitor voltage rating**
```json
// WRONG for 180V rail:
{"refDes": "C1", "type": "capacitor", "value": "100uF/50V"}  // Will explode!
```
**FIX: Use properly rated capacitor**
```json
{"refDes": "C1", "type": "capacitor", "value": "100uF/400V"}
```
