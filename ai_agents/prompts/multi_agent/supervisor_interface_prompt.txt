================================================================================
## INTERFACE DESIGN RULES
================================================================================

### RULE 1: Every Module Defines Its Boundary
- List ALL inputs the module needs to function
- List ALL outputs the module provides
- Include power requirements (VCC, GND, etc.)

### RULE 2: Power Flows From Supply to Consumers
- One module (usually "Power_Supply") provides power rails
- All other modules consume power
- Define voltage and current requirements

### RULE 3: Signals Connect Modules
- Controller modules output control signals
- Driver/actuator modules receive control signals
- Sensor modules provide feedback signals

### RULE 4: Interface Names Must Be Unique and Descriptive
- Use clear names: PWM_CH1 not PWM, VCC_12V not VCC
- Different voltage rails need different names
- Signal names should indicate function

### RULE 5: Types Must Be Specified
- "power": voltage rails (VCC, 12V, etc.)
- "ground": ground connections
- "digital": on/off signals
- "analog": variable voltage signals
- "pwm": pulse width modulation signals

================================================================================
## WHAT THIS ENABLES
================================================================================

The interface contract allows:
1. Each module to be designed independently
2. Modules to be validated against their interfaces
3. Integration to connect modules via interfaces
4. Clear responsibility boundaries

Return ONLY the JSON object.
