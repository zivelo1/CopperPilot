================================================================================
## INTEGRATION RULES
================================================================================

### RULE 1: Power Distribution
- Identify the power source module
- Connect all power rails to consuming modules
- Ensure GND is common to all modules

### RULE 2: Signal Routing
- Connect output signals to corresponding input signals
- Use interface names as net names
- Ensure signal integrity (no floating interface signals)

### RULE 3: Add System-Level Components
- Add bulk capacitors for power distribution
- Add main connector if not present
- Add test points for debugging

### RULE 4: Preserve Module Internals
- Don't modify internal module wiring
- Only add inter-module connections
- Respect module boundaries

### RULE 5: Global vs Local Nets
- Power nets (VCC, GND) are global
- Interface signals are global
- Internal module nets are local (prefix with module name)

### RULE 6: PIN REFERENCE REJECTION
NEVER create a net whose name looks like a component pin reference (e.g., "U2.6", "R10.2").
Net names must be descriptive signal names (e.g., "PWM_CH1", "VOUT_12V").
If you encounter a pin reference as a net name, this is an ERROR — replace it with the
actual signal name from the module's pinNetMapping.

### RULE 7: INTERFACE COMPLETENESS
After integration, verify that every output signal from every module connects to at least one
input signal in another module. Unconnected interface outputs indicate missing wiring.
List any unconnected outputs in "integration_notes" with a clear warning.

================================================================================
## WHAT TO CHECK
================================================================================

1. All interface outputs have corresponding inputs
2. Power reaches all modules
3. Ground is common
4. No orphaned modules
5. Signal flow makes sense (controller → driver)

Return ONLY the JSON object.
