## CRITICAL CONNECTION LAWS - YOU MUST OBEY ALL
================================================================================

### LAW 1: Every IC Must Have Power and Ground
For EVERY IC component (U1, U2, etc.):
- Find the VCC/VDD/V+ pin → connect to appropriate power rail
- Find the GND/VSS/V- pin → connect to GND
- NEVER leave an IC without power and ground!

Example:
- U1.VCC → VCC
- U1.GND → GND

### LAW 2: Every Pin Must Be Assigned to a Net
- EVERY pin of EVERY component must appear in pinNetMapping
- No pin can be left unassigned
- For unused pins, use NC_<ref>_<pin> format

Example:
- U1.NC → NC_U1_NC (no-connect pins)

### LAW 3: Power Rails Need Multiple Connections
- VCC must connect to at least 2 different pins
- GND must connect to at least 2 different pins
- This ensures bypass capacitors are connected

Example:
- VCC connects to: U1.VCC, C1.1, C2.1
- GND connects to: U1.GND, C1.2, C2.2

### LAW 4: TWO-TERMINAL PASSIVES MUST HAVE DIFFERENT NETS (CRITICAL!)
For ANY component with exactly 2 pins (resistor, capacitor, inductor, diode):
- Pin 1 MUST be on a DIFFERENT net than Pin 2
- If both pins are on the same net, NO CURRENT CAN FLOW!

❌ WRONG (shorted - no current flows):
```json
"R1.1": "GND",
"R1.2": "GND"
```

✅ CORRECT (current can flow from VCC through R1 to LED):
```json
"R1.1": "VCC",
"R1.2": "LED_ANODE"
```

❌ WRONG (bypass cap shorted):
```json
"C1.1": "GND",
"C1.2": "GND"
```

✅ CORRECT (bypass cap between VCC and GND):
```json
"C1.1": "VCC",
"C1.2": "GND"
```

VIOLATING LAW 4 WILL BREAK THE CIRCUIT!

### LAW 5: NET NAMES MUST NEVER BE PIN REFERENCES
Net names must NEVER be component pin references (e.g., "Q1.2", "U3.7").
A snubber or protection component must reuse the SAME net names as the component
it connects across, not create new nets named after pin references.

WRONG: "C_snub.1": "Q1.2", "C_snub.2": "Q1.3"
CORRECT: "C_snub.1": "HV_RAIL", "C_snub.2": "BRIDGE_A"  (reuse Q1's actual nets)

### LAW 6: TRANSFORMER WINDINGS MUST BE CONNECTED
Every transformer winding MUST connect to at least one active component on each terminal.
A transformer with floating windings is electrically useless.

### LAW 7: Interface Signals Must Be Exposed
If the interface contract specifies an output signal:
- Create a net with that exact name
- Connect the appropriate component pin to it

Example: If interface has output "PWM_OUT":
```json
"U1.3": "PWM_OUT"
```

### LAW 8: INTERNAL NET CONNECTIVITY
Every net that is NOT an interface signal and NOT prefixed with NC_ MUST connect to at least 2 pins.
A net connected to only 1 pin means a broken signal path.
Before outputting, count connections per net and verify this rule.

### LAW 9: Signal Flow Must Make Sense
- Power flows from supply to load
- Signals flow from output to input
- Current flows through resistors (one end to another)
- Capacitors connect between power and ground (for bypass)
- LEDs need current-limiting resistors in series

## COMMON COMPONENT WIRING PATTERNS
================================================================================

### Voltage Regulator (LM7805, LM317, etc.)
```json
"U1.1": "VIN",        // Input voltage
"U1.2": "GND",        // Ground
"U1.3": "VCC_5V"      // Regulated output
```

### Bypass Capacitor
```json
"C1.1": "VCC",        // Power rail
"C1.2": "GND"         // Ground
```

### Pull-up Resistor
```json
"R1.1": "VCC",        // Power rail
"R1.2": "SIGNAL"      // Signal being pulled up
```

### Pull-down Resistor
```json
"R1.1": "SIGNAL",     // Signal being pulled down
"R1.2": "GND"         // Ground
```

### Current Limiting Resistor (for LED)
```json
"R1.1": "VCC",        // Power source
"R1.2": "LED_ANODE",  // Connects to LED anode
"D1.1": "LED_ANODE",  // LED anode (from resistor)
"D1.2": "GND"         // LED cathode to ground
```

### MOSFET (N-channel)
```json
"Q1.1": "GATE_SIGNAL", // Gate (control)
"Q1.2": "DRAIN_NET",   // Drain (to load)
"Q1.3": "GND"          // Source (usually ground for N-ch)
```

### MOSFET with Gate Resistor (IMPORTANT - LAW 4 applies!)
Gate resistors limit inrush current to the gate. They MUST have different nets on each pin!

❌ WRONG (gate resistor shorted - breaks LAW 4):
```json
"R_GATE.1": "Q1_GATE",  // Both pins on same net!
"R_GATE.2": "Q1_GATE",  // SHORTED - no current limit!
"Q1.1": "Q1_GATE"
```

✅ CORRECT (gate resistor limits current from driver to gate):
```json
"R_GATE.1": "GATE_DRIVE",  // Input from gate driver
"R_GATE.2": "Q1_GATE",     // Output to MOSFET gate
"Q1.1": "Q1_GATE"          // MOSFET gate pin
```

### H-Bridge MOSFET Configuration
For H-bridge with 4 MOSFETs (Q1-Q4) and gate resistors (R1-R4):
```json
// Gate resistors - each bridges driver signal to MOSFET gate
"R1.1": "HI_A_DRIVE", "R1.2": "Q1_GATE",  // Q1 gate resistor
"R2.1": "LO_A_DRIVE", "R2.2": "Q2_GATE",  // Q2 gate resistor
"R3.1": "HI_B_DRIVE", "R3.2": "Q3_GATE",  // Q3 gate resistor
"R4.1": "LO_B_DRIVE", "R4.2": "Q4_GATE",  // Q4 gate resistor
// MOSFETs
"Q1.1": "Q1_GATE", "Q1.2": "HV_RAIL", "Q1.3": "BRIDGE_A",
"Q2.1": "Q2_GATE", "Q2.2": "BRIDGE_A", "Q2.3": "GND",
"Q3.1": "Q3_GATE", "Q3.2": "HV_RAIL", "Q3.3": "BRIDGE_B",
"Q4.1": "Q4_GATE", "Q4.2": "BRIDGE_B", "Q4.3": "GND"
```
